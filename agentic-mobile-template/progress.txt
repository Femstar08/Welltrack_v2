## Codebase Patterns
- Check-in flow uses PageView with NeverScrollableScrollPhysics + PageController for step navigation
- State management via StateNotifier + family provider (keyed by profileId)
- Supabase upsert uses `onConflict: 'profile_id,checkin_date'` for idempotent check-in saves
- Navigation: `context.go('/daily-coach/plan')` after check-in completion, `context.push('/daily-coach/checkin')` from dashboard
- ref.listen previous param is nullable (MorningCheckInState?) — always null-check before accessing
- Route `/daily-coach/checkin` reads profileId from `activeProfileIdProvider`
- Vitality step (morning erection + weekly quality) is in its own widget `checkin_step_vitality.dart` — currently removed from flow, will be re-added by US-007/US-008
- Flutter analyze: run via `cmd.exe /c "flutter analyze"` on WSL2 — returns infos/warnings but 0 errors is the target
- PrescriptionEngine is pure Dart (no async/Supabase) — provider assembles PrescriptionInput from real data, engine evaluates deterministically
- Recovery score drives base PlanType (push/normal/easy/rest); check-in feeling/schedule override the base type
- Adding a new PrescriptionScenario enum value requires updating: fallback_narratives.dart, todays_focus_tip.dart, daily_coach_card.dart (exhaustive switches)
- Use `entity.copyWith()` instead of constructing new DailyPrescriptionEntity when merging AI fields — preserves all fields including new ones
- Bedtime formula: `(wakeHour + 24 - 7) % 24` clamped [21,23] — NOT `wakeHour - 1`
- Use `ref.invalidate(provider)` for cross-provider refresh (e.g. check-in → plan screen)
- Card + InkWell: set `clipBehavior: Clip.antiAlias` on Card for proper ripple containment
- Scenario nudge cards use Hive box `scenario_nudge_dismissals` with key `YYYY-MM-DD:scenarioName` — auto-resets daily
- ScenarioNudgeCard factory constructors: `.tired()`, `.behindOnSteps()`, `.weightStalling()` — prefer these over manual construction
