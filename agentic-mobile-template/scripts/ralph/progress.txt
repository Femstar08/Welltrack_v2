## Codebase Patterns
- Module tiles come from enabledModulesProvider which reads moduleConfigsProvider
- moduleConfigsProvider loads from wt_profile_modules table, falls back to WellTrackModule.values defaults
- WellTrackModule enum in module_metadata.dart controls which modules exist
- defaultEnabled flag on each enum value controls initial visibility
- Dashboard renders tiles via SecondaryModulesList reading dashboardProvider.tiles
- Settings screen is at lib/features/settings/presentation/settings_screen.dart
