## Codebase Patterns
- Use `cmd.exe /c` to run Flutter commands on WSL2
- Must pass `--dart-define-from-file=.env` for Supabase credentials
- All navigation uses GoRouter — NEVER use Navigator.pushNamed()
- Dashboard route is `/` not `/dashboard` — use `context.go('/')`
- `wt_users` has NO email column — get email from `auth.currentUser`
- All Supabase calls must go through repository classes
- Riverpod providers: use .family keyed by profileId for user-scoped state
- After mutations, invalidate downstream providers with ref.invalidate()
- AI calls go through ai_orchestrator_service.dart — never direct from screens
- Error handling: wrap async in try/catch with AppLogger
- All entities use Freezed for immutability
- Use context.push() for stack navigation, context.go() for replacement
- All screens pushed via context.push() MUST have AppBar with back button
- PrescriptionEngine is static class, not a provider
- Recovery score accessed via insightsProvider, not standalone provider
- `FreemiumGate` internally uses Navigator.push — new widgets should use `context.push('/paywall')` directly
- Freemium gates use `currentPlanTierProvider` (FutureProvider)
- `onboarding_completed` is on `wt_users`, NOT `wt_profiles`
- minSdk = 28, AGP 8.11.1, Kotlin 2.2.20
- GarminAttributionWidget/StravaAttributionWidget use `visible` param (default true)
- healthConnectionsProvider is family keyed by profileId, returns HealthConnectionsState
